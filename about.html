1. What it optimizes

You pick a goal (a reward type, all rewards, or a target floor). The optimizer searches how to spend your Archaeology Level points across the stats and returns the allocation with the best expected value for that goal. A separate simulation then replays top builds to check the estimate.

The core measure is reward per stamina spent. A run ends when stamina runs out, so the build that earns the most for each point of stamina wins. Two things make a point of stamina go further and so raise this number: stamina refunds (which give back part of what a hit costs) and auto-crosshairs (free hits that cost no stamina). Attack speed does not change it; see Ticks versus real time.

These are the seven things you spend points on (Strength, Agility, and so on). Points come from your Archaeology Level and are shared across all of them. Each stat has a cap. Divinity needs Ascension 1, Corruption needs Ascension 2. Per point effects:

4. Damage per hit

Flat bonuses are added first, then the percent multipliers:

damage = (base + 1*STR + 2*DIV) * (1 + 0.01*STR + 0.06*COR)

The crit tiers are conditional. A hit can crit; only a crit can become a super crit; only a super crit can become an ultra crit. With chances cc, sc, uc and multipliers cd, sd, ud:

cc = base_cc + 0.01*AGI + 0.02*LUCK        (capped at 99.5%)
cd = base_cd + 0.03*STR
sc = base_sc + 0.02*DIV
sd, uc, ud come from your stats

The average damage multiplier of one hit is the sum over the four outcomes:

m = (1-cc)
  + cc*(1-sc)*cd
  + cc*sc*(1-uc)*cd*sd
  + cc*sc*uc*cd*sd*ud

expected hit damage = damage * m   (before armor)

Worked example: with crit 66.25%, super 60.75%, ultra 22.5% and multipliers 4.21x, 3.11x, 4.84x, an ultra crit lands on 0.6625*0.6075*0.225 = 9.05% of hits and deals 4.21*3.11*4.84 = 63.4x damage.

5. Armor and penetration

Armor subtracts a flat amount from every hit. Perception adds flat penetration, Intellect adds percent penetration that scales the flat value:

pen = (base_pen + 2*PER) * (1 + 0.03*INT)
effective armor a = max(0, block_armor - pen)
damage to block per hit = max(0, hit_damage - a)

A hit that cannot exceed the armor deals nothing but still spends the tick. A block no hit can exceed is unbreakable and is skipped.

6. Breaking a block

Each hit lands one of the four crit outcomes, then loses the armor amount. The expected number of hits to break a block of HP h treats a blocked hit (zero damage) as a wasted tick:

E(h) = ( 1 + sum_i p_i * E(h - d_i) ) / (1 - p0)

where d_i and p_i are the post-armor damage and probability of each hitting outcome, and p0 is the chance a hit is fully blocked. For blocks that take many hits the tool uses the fast estimate E(h) = h / average hit damage, which already discounts blocked ticks.

7. Rewards and mods

Every block gives experience. Non dirt blocks also give loot. The goals can mix experience and loot, so the model needs both on one scale and a fair value per block.

Putting XP and loot on one scale

Experience and loot fragments are different currencies, and the game never states a rate between them. A goal like "all rewards" has to add them, so the model makes one neutral choice and is open about it. Two things set the scale, and the game answers the first for us.

  • How rarities compare. Every block already carries an experience reward, and that number encodes how much the game rewards each rarity: a mythic block gives about 50 times a common's experience. We reuse that scaling for loot, so the worth of a mythic fragment against a common one is the game's own number, not a guess.
  • Loot against experience overall. The game gives no rate, so the only unbiased choice is equal weight: value a block's loot at the same experience the game paired with it on that block.
loot value of a block = its experience   (times a global loot weight, default 1)

A Tier 1 common gives 0.15 experience and a fragment, so its loot is valued at 0.15 too. A mythic at 7.5 experience has loot worth 7.5. Dirt drops no loot, so it adds only its experience. The block pays its full experience to the experience total and an equal amount to the loot total for its rarity.

Why this is fair: the two rewards are treated as equal partners, so a block that drops both is worth twice one that drops only experience, which is correct, you leave with two rewards instead of one. The single judgment call, equal weight, is one setting (Loot value vs XP, default 1): leave it at 1 for a neutral mix, raise it if loot matters more to you, lower it if less. Marking a rarity worthless sets its loot to zero. Intellect then scales the experience total, Perception the loot total.

Mods

When a floor loads, every block rolls its mods and keeps them. A block can carry more than one. A mod adds its multiplier to the block reward only when it rolls, so the expected reward uses the mod chance:

chance:   xp mod   = base + 0.003*INT + 0.002*LUCK
          loot mod = base + 0.003*PER + 0.002*LUCK
gain:     multiplier when it rolls, scaled by Corruption: gain * (1 + 0.01*COR)

expected block reward = base_value * gain_stat * (1 + (mod_gain - 1) * mod_chance)

Because mods are fixed to the block at spawn, a block killed by a crosshair pays the same reward as one killed by the player.

Gleaming floors

A floor is gleaming with a set chance, which multiplies all of that floor's rewards (mods included):

reward factor = 1 + gleaming_chance * (gleaming_multiplier - 1)

8. Stamina and run length

Max stamina, after Agility and the Corruption penalty:

stamina = (base + 5*AGI) * (1 - 0.03*COR)

A run spends stamina floor by floor. A block's stamina cost is its expected hits minus any stamina a stamina mod refunds, so refunds make a block cheaper and the run reach further. When the remaining stamina cannot pay for the next floor, the run stops part way through it. The score is total reward divided by total stamina consumed, shown as value per stamina. A build that earns the same reward for less stamina scores higher, which is the point.

9. Abilities and Instacharge

Three abilities can auto fire when off cooldown:

  • Enrage: the next few swings gain damage and crit damage.
  • Flurry: restores stamina on cast and doubles attack speed for its swings.
  • Quake: the next few swings also deal a share of damage to every block on the floor.

Cooldowns run on real time and do not change with attack speed. The buffed swings happen at the tick rate, so faster attack speed spends the fixed number of charges sooner. Instacharge is a chance to refund a cooldown on use, modeled as a shorter effective cooldown:

effective cooldown = cooldown * (1 - instacharge_chance)

10. Crosshairs

Crosshairs appear in real time, not on the tick clock. Each live block on the floor independently has about a 2.175% chance every 1.005 seconds to spawn one. More blocks on screen means more crosshairs. This rate is a community measurement (see Sources) and can be edited in the app's settings.

Only auto-tap crosshairs fire for an idle player; the rest need a manual tap and are ignored. An auto crosshair fires one regular hit about 2 seconds after it appears. If the block dies first, the crosshair is wasted. Gold is decided when the crosshair spawns:

gold chance = base + 0.005*LUCK
a gold crosshair multiplies its hit by the gold multiplier

A crosshair hit uses the same crit ladder as a normal hit. Because crosshair hits are free, they let the player clear a floor in fewer ticks. The optimizer folds this into the score as an average per floor:

M = 1 + ( (N+1) / (2*v) ) * lambda * t * (hc / hp)

N      = blocks on the floor
v      = attacks per second
lambda = 0.02175 / 1.005   (crosshairs per block per second)
t      = auto-tap fraction (base + 0.02*DIV)
hc, hp = expected crosshair hit and player hit damage (hc includes the gold factor)

The floor's stamina cost is divided by M, which raises reward per stamina and lets the run reach deeper, because crosshair hits add progress and rewards without spending any stamina. The simulation models crosshairs exactly instead, including the 2 second delay and wasted shots.

11. Ticks versus real time

Two clocks run at once and the model keeps them apart.

Tick clockReal time clock
Attacks, stamina spend, block damage.Crosshair spawns, the 2 second crosshair delay, ability cooldowns.
One tick is one swing and one stamina.One real second is always one second.

Attack speed sets how many ticks happen per real second. It does not change stamina efficiency: a build clears the same blocks for the same stamina at any speed. Speed only changes how crosshairs and ability cooldowns line up against ticks, because a faster run spends less real time and so sees fewer crosshair spawns. For this reason attack speed is excluded from the per-stamina score and only matters inside the crosshair and ability terms.

12. The optimizer

The problem is its size. Level points can be split across seven stats under their caps (Strength and Agility 50, Perception, Intellect and Luck 25, Divinity and Corruption 10). At high levels that is millions of possible builds, too many to score one by one on a phone. So the tool uses a guided search by default and keeps a full search as a check.

Guided search (default)

It begins from a handful of sensible builds (all points in one stat, an even split, and a few random splits) and scores them. Then it repeatedly tries moving a single point from one stat to another and keeps any move that scores higher, until no single move helps. It runs this from each starting build, and now and then jumps to a fresh random build, so it does not settle on a build that is good but not the best. This reaches the top build while scoring only a few thousand builds instead of millions.

Full search (the "Thorough" toggle)

This scores every legal build, so it is guaranteed to find the best one. It runs only when the count is small enough to finish quickly; otherwise it hands off to the guided search.

How we know the guided search is right

It was checked against the full search on 40 random setups covering every ascension, goal, and tolerance setting. The guided search returned the same best build in all 40. For a hard guarantee on one small case, switch on Thorough and compare.

Goals and tolerance

With a secondary goal and a tolerance, the tool first keeps every build within the tolerance of the best primary score, then ranks those by the secondary goal. This is how, for example, it picks the build with the most experience among the builds that reach close to the best loot.

13. The simulation

The simulation replays a build many times with random crits, mod rolls, block layouts, abilities, and crosshairs. It re-ranks the top builds from the search and reports, per build: average reward per stamina, the floor distribution (10th, 50th and 90th percentile), and, for a target floor goal, the percent of runs that reach it. It is the exact model and the right tie breaker when two builds score close.

15. What is sourced, what is our method, what is estimated

Three kinds of input feed this tool. They are kept apart here so you can judge each on its own and trust the parts that deserve it.

Grounded in game data

  • Block HP, armor, experience, fragment counts, floor windows, per-floor spawn rates, and boss floors, from the wiki.
  • Stat effects and caps, from the stat panel and wiki.
  • The damage formula, the crit, super and ultra ladder, armor and penetration, stamina, stamina refunds, and the gleaming multiplier. These are game mechanics; the math layered on them is standard probability.
  • Mods roll when a floor loads and stay on the block, so the killer (player or crosshair) does not change the reward.

Our modeling choices

  • One neutral bridge between experience and loot. The game gives no rate between them, so we value a block's loot at the block's own experience (equal weight, default 1), take the rarity scaling from the game's experience numbers, and expose the experience-to-loot weight as a single setting. Reward is scored per stamina consumed, net of stamina-mod refunds.
  • Speed is left out of efficiency. Attack speed changes attacks per second, not reward per stamina, so it only enters through the real-time crosshair and ability terms.
  • Averages for the search, exact replay for the simulation. Expected hits, ability uptime, and the crosshair term are averages so the search can score many builds quickly. The simulation runs the exact rules and is the better number when the two disagree.
  • The guided search is a search, not a proof. It matched the full search on every tested case; use Thorough for a guarantee on one case.
  • Idle play. Only auto-tap crosshairs are counted; manual taps are ignored.

Estimated or unconfirmed

  • Crosshair spawn rate (about 2.175% per block every 1.005 seconds) is a community measurement, not an official figure. It sets how much crosshairs are worth, so it is an editable setting. The 2 second fire delay is observed.
  • Crosshair details. We assume auto crosshairs do not consume ability charges. The gold multiplier comes from your stats; its default is a placeholder.
  • The crosshair value inside the optimizer stacks several averages (live blocks per floor, full use of crosshair damage, armor left out of the damage ratio) on top of the estimated spawn rate. It is the least grounded number that can change which build is recommended. When in doubt, trust the simulation.
  • Polychrome and infernal cards are recordable but treated as neutral until their multipliers are confirmed.
